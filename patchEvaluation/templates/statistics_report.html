🔍 スキップ理由の詳細説明

NO_INTERACTION_LOG・EMPTY_INTERACTION_LOG

APRシステムがログを生成できていない、またはログが空の状態。APRツールの実行自体が失敗している可能性があります。

ALL_MODIFICATIONS_NULL・NO_MODIFICATION_PROPERTY

APRシステムが修正案を生成できなかった状態。問題の難易度が高い、またはAPRツールの能力を超えている可能性があります。

INVESTIGATION_PHASE

APRシステムが問題を理解するための情報収集段階で終了。より多くの情報が必要な複雑な問題の可能性があります。

FINAL_TURN_NO_MODIFICATION

途中まで修正案を生成していたが、最終的に修正案を提出せずに終了。部分的な解決に留まった状態です。

🔍 評価結果の計算方法と判定基準

📊 割合計算式

完全一致率: (IDENTICAL件数 ÷ 評価実施総数) × 100
意味的同等率: (SEMANTICALLY_EQUIVALENT件数 ÷ 評価実施総数) × 100
妥当だが異なる率: (PLAUSIBLE_BUT_DIFFERENT件数 ÷ 評価実施総数) × 100
不正確率: (INCORRECT件数 ÷ 評価実施総数) × 100
スキップ率: (SKIPPED件数 ÷ 全データセットエントリー数) × 100
※ 評価実施総数 = IDENTICAL + SEMANTICALLY_EQUIVALENT + PLAUSIBLE_BUT_DIFFERENT + INCORRECT
※ スキップ率のみ全データセット数を分母とし、その他は実際に評価を実施した件数を分母とする

⚖️ LLM評価判定基準

各修正は以下の5段階で評価されます。上位2つ（IDENTICAL + SEMANTICALLY_EQUIVALENT）が正解として扱われます。

🎯
IDENTICAL（完全一致）
★★★★★ 最高品質
Ground Truthと文字通り完全に同一の修正
  • コード構造、変数名、実装方法が全て一致
  • 空白文字やコメントを除いて完全に同じ
  • 人間が書いた正解と全く同じコードを生成
📝 具体例:
Ground Truth: if (x != null) return x.getValue();
AI修正: if (x != null) return x.getValue();
→ 完全一致！
✅
SEMANTICALLY_EQUIVALENT（意味的同等）
★★★★☆ 正解レベル
異なる実装だが、機能的に完全に同等の修正
  • 同じ問題を異なるアプローチで解決
  • 実行結果や動作が Ground Truth と同じ
  • コーディングスタイルや手法の違いは許容
  • 15の意味的同等ルール（R1-R15）のいずれかに該当
📝 具体例:
Ground Truth: return x != null ? x.getValue() : null;
AI修正: if (x != null) return x.getValue(); else return null;
→ 意味的に同等！（三項演算子 vs if文）
🟡
PLAUSIBLE_BUT_DIFFERENT（妥当だが異なる）
★★★☆☆ 部分正解レベル
妥当性はあるが、最適解ではない修正
  • 部分的に問題を解決する代替的なアプローチ
  • 構文的に正しく、論理的にも妥当
  • しかし Ground Truth とは異なる問題解釈
  • 追加の手直しや改善が必要な場合がある
📝 具体例:
Ground Truth: list.clear(); list.addAll(newItems);
AI修正: list = new ArrayList<>(newItems);
→ 妥当だが異なる！（参照の扱いが違う）
❌
INCORRECT（不正確な修正）
★☆☆☆☆ 不正解レベル
問題を解決していない、または有害な修正
  • 問題を適切に解決していない
  • 新たなバグや問題を引き起こす可能性
  • 構文エラーや論理的な誤りを含む
  • 要求仕様を満たしていない
📝 具体例:
Ground Truth: if (x != null) return x.getValue();
AI修正: return x.getValue();
→ 不正確！（NullPointerExceptionのリスク）
⏭️
SKIPPED（評価スキップ）
－ 評価対象外
技術的な理由で評価できなかった場合
  • interaction_log（対話ログ）が存在しない
  • APRログの構造解析に失敗
  • 必要なファイル情報が不足
  • LLM評価に必要な前提条件が未満たし
🏆 正解の定義
正解 = IDENTICAL + SEMANTICALLY_EQUIVALENT
✅ IDENTICAL: Ground Truthと完全一致
✅ SEMANTICALLY_EQUIVALENT: 意味的に同等（異なる実装でも機能的に同じ）
⚠️ PLAUSIBLE_BUT_DIFFERENT: 妥当だが最適解ではない（部分正解）
❌ INCORRECT: 不正確または有害
⏭️ SKIPPED: 評価不可能

🔄 評価プロセス

  1. APRログ解析: interaction_log から対話内容と修正履歴を抽出
  2. 最終修正特定: 最後に適用された修正を特定
  3. Ground Truth比較: データセットの正解修正との詳細比較
  4. コンテキスト準備: 問題説明、関連ファイル、修正内容を整理
  5. LLM評価実行: GPTモデルによる修正内容の正確性・妥当性評価
  6. 5段階分類: IDENTICAL/SEMANTICALLY_EQUIVALENT/PLAUSIBLE_BUT_DIFFERENT/INCORRECT/SKIPPEDに分類
  7. 統計集計: 各評価レベルの件数と割合を計算し、正確性・妥当性の詳細分析を実施
